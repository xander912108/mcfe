# Agent Instructions

## Validation

- After completing any code or content changes, run `npm run lint` before finishing the task.
- Treat ESLint errors as blocking: fix errors introduced by the task before presenting the work as complete.
- The project currently has existing lint warnings, so do not treat the historical warning count as a merge blocker by itself.
- Do not introduce new lint warnings. If warnings remain after the task, clearly document whether they are pre-existing, unrelated, or caused by an environment limitation.
- Do not present work as ready to merge unless lint has 0 errors and any remaining warnings are explicitly documented.
